What is responsive web design?

Mobile first

Mobile version of site often degradation, try and make everything fit on tiny screen - basically degrading a site without ruining everything. Better to design mobile version of a site first, and then going from there

Reasons: mobile is exploding, mobile forces us to focus on most important features and clean design, mobile offers interesting interactions.

That means before starting, good to have an idea what phone experiences look like and what users need

Mobile first in HTML means: define default for tiny screens, and then add (at end of document) for @media (min-width: 768px) - for tablets and @media (min-width: 1200px) - for desktops

In HTML-doc, add in beginning <meta name="viewport" content="width=device-width, initial-scale=1">

Types of layout

Static: Requires horizontal scrolling if window too small (and content can't be squeezed because width is defined). Answer for smaller screens: creating a separate site for lower resolutions

Adaptive: Defined layouts for different resolutions - basically a series of static layouts. Within each layout, resizing the window does not change the layout. Use CSS feature called media query to load different style sheets for different browsers. Necessary to design very device-specific

Liquid/fluid: Scaling width of design parts relative to the window. Width always expressed in percentages. Default setting for width in HTML is fluid. Can be problematic, because it squeezes big content on small devices

Responsive: Defined layouts for different resolutions. Within each layout, design is liquid and resizes width relative to changing window size - so combining user-centered approach with good maintainance. Using percentage and max-width and media queries

Basic things for responsive web design

  • For responsive design, use media queries
  • Use percentages for width. Percentage of child-element orientiert sich am Elternelement. Kinderelement, das eiche Größe hat wie Elternelement bekommt 100% (will be automatically adjusted), columns z.B. 50%, 33.33% oder 25% (wenn man überall columns haben will)
  • Ensure readability for all devices
  • Make images and iframes responsive

Media queries

Adapt to browser sizes

Media queries can be used for all types of layout.

  • Adaptive design - adapting to a specific set of devices or browser widths, for example three layouts for desktop, tablet, and phone - layout jumps at certain width. So layout is optimized for three specific widths
  • When user visits website, browser collects information about user: their device, browser width, device orientation, screen resolution. Media queries instruct browser to apply CSS to our website only when screen has reached a certain size. Tell browser which CSS to use at which browser width
@media (max-width:600px) { - @media is catch-all term for addressing devices
body {
background: red; }
}

@media (max-width: 601px) { - for resolutions 600px and lower.
body {
background: blue; }
}

@media (max-width: 601px) { - for resolutions 601px and higher. Don't use 600px twice, this will create bugs
body {
background: blue; }
}

Modifiers and operators

Modifiers for media queries

  • width
  • height, min-height, max-height
  • device-width
  • braille
  • handheld
  • Check out more modifiers

Operators

  • and: combining multiple media features together. @media (min-width: 700px) and (orientation: landscape) - so only applies if display is in landscape and over 700px width. @media (min-width 601px) and (max-width: 700px) { ... }: this means between 601 and 700px
  • comma-separated lists - "or": if any of the media queries returns true, styles or style sheets get applied, so in above example, would be for min-width of 700px or orientation landscape.
  • not
  • only...

Change types of media, like print

Think about what user needs in print, and what not

@media print {
body {
background:none;
}

img {
display:none;
}
}

How to use media queries and breakpoints

In CSS

  • Media queries added at end of CSS stylesheet
  • Think about which code can be reused for which devices

Breakpoints

A breakpoint is a modifier on a media query that tell browser which CSS to use for different devices and widths

How many breakpoints do we need?

  • Landscape of devices constantly changing. Goal not to create 12 different versions to cover as many devices as possible. Responsive website tries to be balanced - so adding breakpoints for truly different user experiences.
  • Choose according to content - when does it look weird, when does text become hard to read?
  • With mobile-first approach, we should have 2 breakpoints: one for tablet (standard breakpoint 768px) and one for desktop (940px)

Make typset, images, and iframes responsive

Responsive Typography

Use ems and rems

Most important: legibility.

  • Standard font size on web is 16px font size - so much larger than in print.
  • Optimal length for line is between 45-75 characters
  • Pixel is fixed size and therefore far too static. Making font size responsive: with ems and rems.
  • Em are scalable units - scaling in relation to parent element and thereby creating relationships (defining 12px or 16px as default in html; and then 3em for headings, 1em for text etc.). Parent relationship very strong with ems, if any div is given a font size, child element in div will accept this as their new base font size
  • Rem: here font size is based on root element: html {font-size: 16px;}, h1 {font-size: 3rem;}

Scaling in relation to what?

  • Rems very helpful, but if all elements depend on root element, all site elements are interrelated in size.
  • Modules: For responsive sites, better to create modular elements that can be scaled individually. So think about which site parts should form modules, which elements make sense as parents

Responsive images

Make images web-friendly

Export images from photoshop and check if it has right size and resolution

  • Right size: smaller than 100KB and always smaller than 1MB
  • Dimensions: no larger than 1000px wide (but as large as needed)
  • Resolution: should be 72 ppi
  • Web readable format: jpg, png, gif, svg. JPG good for photos.
  • Usually stick to jpeg and png. In GIMP, go to "export as" and select "by file extension", choose png or jpeg, hier kann das Bild auch komprimiert werden - dafür vor dem Abspeichern image quality geringer machen (bisschen reicht schon), geht nur bei jpg.

Make images responsive

  • Nest img tag in <figure> tag.
  • Add classes to figure tag: <figure class="header-img"> or <figure class="thumbnails">
  • In CSS documents, global Styles, add img {width: 100%, height: auto;}. Images then inherit width of parent element where it is nested.
  • In sections for mobile/desktop, add specific element and add max-width and max-height: .top-img img {max-width: 300px; max-height: 300px;} - making them flexible enough

Make them fit

If I have to add a lot of pictures and don't have size to resize all of them, make them background images. Instead of loading image into html, add it in image as CSS.

  • First, in HTML-doc create empty div (wo das image hinkommt) mit class (thumbnail, das auch für andere thumbnails verwendet werden kann) und id (um das spezielle Bild im CSS zu laden)
  • In CSS-doc, load image: #my-img {background-image: url ('link-to-image.jpg');
  • In CSS-doc, define thumbnails:
.thumbnail {
background-position: center; - damit nur die Ecken gecroppt werden
background-size: cover; - das heißt, dass es die volle Höhe und Breite des divs einnimmt
background-repeat: no-repeat; - damit es sich nicht wiederholt
width: 40%; padding-bottom: 40%; - um das Bild quadratisch zu machen; height nicht auch setzen, das würde nicht funktionieren im responsiven Design
float: left;
margin-right: 20px;

Responsive Iframes

Special html tag that allows us to add html doc (hosted by other web site) in html doc

Iframes, taken out, have fixed width and height. Copy iframe in HTML-doc and nest it in <div class="map container">

In CSS, define:

.map-container {
position: relative; (allows us to set iframe to position: absolute)
padding-bottom: 75%; (not about padding, setting relationship between height and width, so aspect ratio. Depending on aspect ratio of map)
height:0; (padding gives it height it needs)
overflow:hidden; (so map does not go over parent-element)

Then define iframe properties:

.map-container iframe {
position: absolute;
top: 0;
left: 0;
width: 100%;
height: 100%;

Tool that does that automatically: Embed-resonsively.com

Coding with boilerplates and style tiles

What is a boilerplate?

Code shell: basic html code for site - makes sense to group sites with similar layout and functions

HTML5 boilerplate: used by most major companies, constantly updated. Comes with modernizr - code to normalize HTML5-code which otherwise does not work with all browsers

Get HTML5 boilerplate here.

Adjust boilerplate

Start with layout CSS. In Boilerplate, two CSS documents.

  • Main.css: where I write my CSS code. Contains some default styles with comments.
  • My styles go on bottom. Start by writing all my HTML elements and classes - first header with navigation, then other elements according to order to html document
  • Set widths: For main containers (header, main, footer), set width to 96% and 2% padding on left and right. For elements inside, give them 100%
  • Set breakpoints: 768px tablet, 940px desktop. Replace media query in main.css and replace it with my media queries: @media screen and (min-width: 768px)
  • Normalize.css: contains browser compatibility styles - best not to edit it

Style tiles

Style tiles are middle point between moodboard (sehr grob) and detailed mockup. Contains typography and interface elements like colors and patterns - creating free-floating visual language for site without getting lost in details of layout (which can change) - so layout remains fluid and adaptable

Coding style tile

In custom styles in main.css, add in beginning section for global styles:

body, html (font-faily, font-size, color)
a (font-weight, color, text-decoration)
p (font-size, line-height)
h1, h2, h3, h4
h1 (text-transform)
h2
h3
h4
.buttons ...

After that, style individual sections

The flex model

What is flexbox?

New layout type introduced with CSS3. Set of CSS properties to allow for more flexible layouts

Flexbox vs. box model

  • With box model, html elements could be either vertically oriented (display:block) or horizontally oriented (with display:inline).
  • Box model works well mainly for desktop versions. For other browsing experiences, certain limitations
  • Flexbox is both vertically and horizontally oriented. Idea of having two axises (x-axis or main axis and y-axis or cross axis)
  • Like text-align for layout elements, both on x and y axis (center align, left align, justify, stretch vertically, top-align)
  • Also easily possible to change order of HTML elements
  • Flexbox supported mainly by new browsers (check on caniuse.com)

Define flexbox elements in HTML and CSS

  • In HTML doc, define section as class="mein-flex-container"
  • Then, define all elements in this container as class="meine-flex-items"
  • In CSS doc, give elements property {display:flex;} - turns element in a flex container (makes its child elements flex-items)
  • body {display:flex;} - makes all body elements flex items
  • For all flex elements, add browser prefixes
display: flex;
display: -webkit-box;
display: -moz-box;
display: - ms-flexbox;
display: -webkit-flex;

Flexbox properties

Justify-content

Centers content on main axis. Determines how to situate flex-items on main axis. If I want to make flex-items bigger, add width. If I give justify-content property to flex-items, div-tags inside will also behave the same way (center or be at start...)

  • justify-content: flex-start; - left-aligning content (at beginning of container)
  • justify-content: flex-end; - right-aligning content
  • justify-content: space-between; - aligning elements in container with even amount of space in between
  • justify-content: space-around; (even amount before, between and after elements)
  • justify-content: initial; - default value
  • justify-content: inherit; - take on positioning of parent-element

Align-items

Aligns elements vertically, or aligns content of items vertically if I add this property to items)

  • align-items: center;
  • align-items: stretch; - elements vertically fill container
  • align-items: flex-start; - on top
  • align-items: flex-end; - on bottom
  • align-items: baseline; - positions elements usually on top of container
  • align-items: initial + inherit

Align-content vs. align-items

  • Align-content has same values as align-items.
  • Align-items is best when I want to align a row of content vertically.
  • Align-content is best when I have wrapped content

Flex-wrap

Tells items to wrap or not to wrap

  • flexwrap: wrap;
  • flexwrap: wrap-reverse; - in reverse order
  • initial + inherit

Flex-direction

Dictates direction of items

  • flex-direction: row; - display items horizontally in a row
  • flex-direction: row-reverse; - in reverse order
  • flex-direction: column; - display items vertically in a column
  • flex-direction: column-reverse;
  • initial + inherit

Flex-basis

  • Sets base width of element (like min-width). If I set flex-basis for container and items, flexbox calculates amount of space left over (not used by items) and redistributes this space using flex-grow or flex-shrink properties.
  • flex-basis: 200px; - setting width
  • flex-basis: number; - unit or percentage
  • flex-basis: auto; - length determined by item's content

Flex-grow and flex-shrink

  • If I have 100px left, I use flex-grow property to determine how remaining space is redistributed so that elements fill up full width of container. If I set all elements to flex-grow: 1, equal distribution. If I set only one item to flex-grow 1, this item will grow by 100px, other two would stay the same.
  • With flex-shrink, I manage width if container is too small for divs. With flex-shrink I can set rate at which items shrink to fit inside container.
  • flex-grow: number - default is 0
  • flex-shrink: number; - default is 1
  • initial + inherit

Use flexbox to create layout

Not rely entirely on flexbox, as it is not used by all browsers. Flex useful to divide up space between elements in a layout

Flex-flow

  • Combines values for flex-direction and flex-wrap in one line of CSS. Default: row nowrap
  • Tells container how to display its items: which direction to display flex-items and if they should wrap.
flex-flow: row wrap;

Flex

Property flex: combination for flex-grow, flex-shrink, and flex-basis

  • flex: 2 100px; - grow by 2, initial length 100px
  • flex: auto;
  • flex: initial; + inherit

Order

Property order: if I have several flex items in same container, I can tell screen in what order to display them. If for desktop, I want a sidebar to be one the right and for tablet on the left, I just have to change number of order it shows up:

media (min-width 768px) {
.sidebar-1 {order: 3;} - on the right
.main-content {order: 1;} - on the left
.sidebar-2 {order: 2;} - in the middle

media (min-width 1200px) {
.sidebar-1 {order: 1;} - on the left
.main-content {order: 2;} - in the middle
.sidebar-2 {order: 3;} - on the right
}

Further combination of properties

Align element both horizontally and vertically

div {justify-content: center; align-items: center;}

Wrap list of 6 evenly sized items

ul.flex-container {
display:flex;
flex-flow: row wrap;
justify-content: space-around;
}

li.flex-item {
width:200px;
height: 200px;
}
