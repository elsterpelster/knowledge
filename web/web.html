Command line / terminal

  • Konsole, Kommandozeile / command prompt, command line, CLI
  • Nutzerendgerät für Eingaben und Anzeigen von Daten, Interface für Steuerung einer Software
  • First computers controlled via punch cards - punching programs into series of cards that were fed into computer via terminal. Replaced by video terminals in 1960s. Small screens in which a series of commands could be typed via keyboard.
  • In 1970s, giant mainframe computers were reduced down to microcomputers, became then smaller and more affordable. But still needed to be controlled via terminal
  • First GUI 1983 on Apple Lisa, successful since 1984 in Macintosh computer. First time using desktop metaphor, possible to interact with computer without using commands
  • Command line speaks directly to computer by Unix commands. Easier and faster than GUI, especially when working with many programs

Git, Github and the command line

Definitions

Git is most popular version control system.

  • Version control: software that tracks any changes that I make to my code files
  • Ensures I have backup of my code and saves copies of code at various stages
  • Makes it possible to work with other developers on same codebase at the same time without interfering with each other

GitHub is a SAS (software as a service), which allows me to host code in the cloud (on server of GitHub)

  • Accessible from everywhere, possible to share with others
  • Other services for hosted Git repositories: Bitbucket, Beanstalk, RhodeCode ... and I can also set up my own Git server

How it works

  • If I work on website, I have folder with some files on my computer. Using the terminal, I can turn folder into repository in Git (local repository on my computer)
  • Create matching repository on GitHub to create backup of my files (remote repository)
  • Sync local with remote repository with "git pull"
  • Edit files in local repository, commit (save) changes to local repository with "git add" and "git commit"
  • Sync local to remote repository again
  • Push changes to remote repository with "git push"

Linux

OS: Software, die Verwendung des Computers ermöglicht.

  • Besteht aus Kernel und zugehöriger Software, die in Form von Distributionen vertrieben wird

Kernel: basic services for all other parts of OS.

  • Interface zwischen Shell und Hardware, ermöglicht Nutzung der Hardware
  • Linux-Kernel kann an sich nichts darstellen, dafür X-Window-System

Shell: outermost part of OS that interacts with user commands

  • Interpreting command line for Kernel
  • BASH: Bourne again Shell

X-Window-System/X: Protokoll, das von Software umgesetzt wird

  • Basic framework for GUI
  • Server/XServer: wird configueriert, connected Hardware

XML und HTML

Evolution HTML: HTML 3.0, 4.0, danach XHTML, jetzt HTML 5.0

  • XML als übergeordnete Sprache/Syntax.
  • XML: textbasiertes Format für den Austausch von strukturierten Informationen (z.B. Texte, Rechnungen, Bücher)
  • XML soll Informationen fürs Internet anwendbar machen und menschenlesbar sein
  • HTML ist vom gleichen Standard (SGML) abgeleitet, ist aber eigentlich formal unabhängig von XML.
  • HTML 4.01 wurde für XML übersetzt, daraus ist neuer Standard XHTML hervorgegangen als gemeinsame Schnittmenge
  • Dieser Dualismus zwischen HTML und XML wurde jetzt beendet mit HTML5

Bootstrap

What is bootstrap?

Bootstrap is most popular front-end framework (component library), open-source toolkit for developing with HTML, CSS and JS.

  • Created in 2011 as Twitter bootstrap
  • Spotify and other big sites built with bootstrap
  • Toolbox which gives you code/classes that make it easy to create things like navigation bars, buttons, sidebar, pagination - and so saves a lot of time for things for which there is no need to start from scratch
  • Good for prototyping/building sth quickly - out of the box results look already pretty decent, can then be customized
  • Developer-friendly, if you are not so much into designing, there are all kinds of pre-built features.
  • Responsive, mobile first.

Why use bootstrap?

  • Very popular, widely supported, lots of online resources
  • You can use and adapt the code you get
  • Relatively easy to learn. There are other front-end frameworks, but transitioning is relatively easy, as they all work in a similar way
  • Compatible with all browsers
  • Compatible with other preprocessers like Sass and Less
    • Sass: Meta-Sprache von CSS. Hilft dabei, Stylesheets schneller zu schreiben und Techniken unkompliziert (und browserübergreifend) einzusätzen. Sass automatisiert bestimmte Aufgaben und vereinfacht Umgang mit Farben und Gestaltungsrastern
    • Regeln werden in .scss Dateien (statt in .css-Dateien) geschrieben, aus denen dann normale .css-Dateien generiert werden.
  • Bootstrap has wide range of themes and templates, can be integrated in other languages, also for building mobile app.

What can be problematic about bootstrap?

  • Bootstrap has lots of code, so would be rather overloaded for simple sites.
  • Uses also lots of classes
  • Many sites use it, and if it is not customized enough, sites look similar (and if too much customization, then sometimes easier to start from scratch)

Get started with bootstrap

  • Get bootstrap from getbootstrap.com.
  • Bootstrap CDN: Also possible to not download, and instead load from external site, but this option will take longer to load. (CDN: content distribution network, large distributed system of servers deployed in multiple data centers across the internet)
  • Bootstrap contains css files and js files, so I need to create my own HTML document: index.html-file in the same folder.
  • On bootstrap website, under Getting started copy basic HTML template into index.html file, very useful for getting started. Ensures to load bootstrap files (css files, jQuery (necessary for bootstrap javascript plugins), and javascript files)
  • Useful to create container-div around my content for adding margin and paddings

Key compontents

Bootstrap grid system

Grid system is broken down into 12 columns, which can be allocated freely in 3x col-md-4, 2x col-md-6, col-md-4 + col-md-8, ... If I want to start a grid, I need to wrap columns in <div class="row">...<div>. Col-class relies on row-class as parent.

<div class="row">
<div class="col-md-6">Column one, half width</div>
<div class="col-md-6">Column two, also half width</div>
</div>

Grid system has 4 classes

  • xs for phones, sm for tablets, md for desktops, and lg for larger desktops. Classes can be combined to create more dynamic layouts
  • Classes scale up, so if I want set same widths for xs and sm, I need to specify only xs.
  • Default settings for xs: never stacking, all others only at certain breakpoints

Images

Adding classes to img to say how they should be displayed

  • <img src="..." alt="..." class="img-circle"> - display round picture.
  • Class="img-rounded", image gets rounded edges
  • Class="img-responsive" to make image responsive, that is to make it fit into its container (saves you the time to do that yourself)

Navigation bar

On getbootstrap.com, copy default code for navigation. All custom styles responsive

Buttons

  • Button classes to change colors, sizes etc. Buttons on getbootstrap.com
  • Colors: btn-default - white button, btn-primary - blue, btn-success - green, ...

Suffixes and features

  • Suffixes for buttons appear for other things like panels, progress bars, labels etc.
  • Same features to make things smaller and bigger etc.

Customization

  • Especially for starting out, good to copy templates and then adapt them und understand them
  • If I want to add my own styles, add custom.css file in css folder, and add this stylesheet in my HTML doc (not replace bootstrap css, just add it)
