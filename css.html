What is CSS?

CSS tells browser how to display html tags

  • CSS: Cascading Style Sheets
  • Cascading: Vererbung – von Schriftart, Schriftfarbe, Schriftgröße auf alle Kinderelemente
  • Sheets: „Folien“, design.css, einzelne Folien, die Design festlegen

Parts of CSS markup

  • CSS selector: h1, p...
  • CSS properties: name: value;
HTML tag {
property1: value;
property 2: value;
}

Example

p{
color: purple;
width: 200 px;
font-size: 24px;
font family: comic sans ms;
}

Most important properties

Many more CSS properties

Schriftart, -größe, -farbe, fett/kursiv

font-family: accepts web safe fonts (preloaded on browser)

  • arial, times, georgia, tahoma..
  • serif, sans-serif

font-size: accepts web safe fonts (preloaded on browser)

  • either in pixels or em (relative text size measurement, very useful for responsive web design)
  • 15px: damit bestimme ich, dass Schriftgröße für jeden Browser 15px ist, selbst wenn der User sich das anders eingestellt hat
  • Besser: 1em (entspricht in Standardbrowsern 16px), oder eben 0.983em.
  • Meiste Browser haben <h1>: 2em vom Body
  • Wenn ich im Body Schriftgröße auf 15x gesetzt habe, dann nehmen alle nachfolgenden Kindelemente (h, cite etc.) 15px als Bezugsgröße an. Wenn Zielgröße für h2 also 21px ist, dann 21/15=1.4em

color:

  • web safe color names (red, green, blue..)
  • or hexadecimal values (for example at W3 schools.) Hex colors always start with #

font-weight:

  • bold, bolder, normal, lighter
  • 100-900

font-style:

  • italic

font-variant:

  • small-caps

text-transform:

  • capitalize, uppercase, LOWERCASE

text-decoration:

  • underline, overline, line-through

Textausrichtung, Zeilenhöhe und -abstand, Wortabstand

text-align:

  • left, right, center, justify

text-indent:

  • for indenting beginning of paragraph
  • px, em, percentage

line-height:

  • Zeilenabstand
  • in px, em, numbers, percentages

word-spacing:

  • between words
  • px, em, and negative values

letter-spacing:

  • within word
  • px, em, and negative values

Colors

2 options for colors

  • Web safe colors, with hex values
  • Hexadecimal values: # + 6 digits or letters
  • Besser in Hexa als mit Farbwörtern, wird sicherer von allen Browsern erkannt

Color models

RGB color model (Red, Green, Blue): #RRGGBB

  • Additive color model: mixes colors by light passing through them
  • #FFFFFF = white. F= each color is turned on 100%.
  • #000000 = black. 0= no color turned on
  • #FF0000 = red

CMYK color model (Cyan, Magenta, Yellow, Black): for print.

  • Subtractive color model - different method, so web colors never perfectly match print colors

Add colors to CSS

Background

  • body {background: powderblue;}
  • background also supports images. background: url('img/flower.png') no-repeat
  • no-repeat: problem if image is tiny. background: url('http://...') repeat
  • repeat-x (along x-axis) and repeat-y (along y-axis)
  • Position image on screen: no-repeat X Y white. Position on x and y axis + surrounding color
  • background: no-repeat 10px 20px white

Borders

  • Necessary properties: thickness, border style, and color
  • main {margin: 20px; border: 5px solid purple;}
  • border style: solid, dashed, dotted, groove, ridge

Height and width

Set height and width of HTML block elements (inline elements does not make much sense)

  • Fixed height/weight: section {margin: 20px; border: 5px dotted #333; width: 300px; height: 500px;} - meaning they stay the same for all screen sizes (not responsive)
  • Not a good idea to set both height and width, else overflow.
  • I can also add {overflow: hidden; overflow: scroll;}

Selectors, classes and IDs

Selektoren

  • Elementselektor: body {...;}, p {...;}
  • Klassenselektor: verfeinern von Klassen. .rot {...} oder p.rot {...} – gilt dann nur für rote Absätze
  • ID-Selektor: spricht ein Element pro Dokument an, dass diese ID hat. Ich kann auch Absätzen eine ID geben. #navi {...}
  • Kontextselektor: spricht Elemente mit bestimmten Eigenschaften an, innere Elemente in Elternelement (z.B. alle Bilder in Datenzellen): td img{...}
  • Pseudoelementen-Selektor: p:first-letter{…} spricht z.B. nur den ersten Buchstaben eines jeden Paragraphen an. Buchstabe wird behandelt, als wäre es ein Element, obwohl da kein Tag ist
  • Attributselektor: spricht alle Elemente an, die bestimmtes Attribut mit Wert xy haben. Input[type=submit]. Damit werden also nicht andere Textfelder erwischt, sondern nur input type submit, d.h. Buttons zum Abschicken.
  • Oder auch Input[placeholder]: Nur inputs, die placeholder-Attribut haben (Wert muss dabei nicht bestimmt werden)

Classes:

General rules with selector h3 {...}, exceptions with classes

  • in CSS: .h3-exc {color: blue;}
  • in HTML: <h3 class="h3-exc">

Beispiel: alle h3 sollen blau werden, aber ein paar rot

  • Dann lege ich h3 {color: blue;} fest
  • und lege eine Klasse für die Ausnahmen an: .h3-ausnahme{color: #FF0000;}
  • und markiere dann im HTML doc die Ausnahme-H3s: <h3 class="h3-ausnahme">Überschrift</h3>

Universalattribut, kann nicht nur einem Element zugeordnet werden

  • Sondern auch anderen HTML-Elementen im Dokument (ein paar p's usw.)
  • Andersrum können HTML-Elemente mehrere Klassen haben.

IDs

ID: Attribute only for one ID (#h3-extra, ...)

  • ID-selector: spricht das eine Element an, das diese ID hat #h3-extra
  • in CSS: #h3{background}
  • Nur ein Element pro ID (gibt nicht 2 #navi)
  • Andersherum können Elemente auch nur eine ID haben

Rules for IDs and Classes

Names for IDs and classes

  • Not start with number
  • Name: say what it does, not how it looks - so "alert" rather than "red" (semantic web, also, more functional)

Speak to certain elements in my class

  • I want to change list items in my inside navigation so that they appear as block elements with pink background and 5px margin between list elements
  • I cannot address the whole inside-navigation, because then I cannot mark the margins (and it would look like a pink block)
  • So I want to address the list items in my inside-navigation
  • First, in my HTML doc I mark inside navigation as a class: <ul class="inside-nav">
  • Then in the CSS doc, I address list items in this class:
.inside-nav li {
list-style-type: none;
display: block;
background: pink;
margin: 5px;
}
  • Parent-element child-element {define;}

Pseudoklasse:

  • a:hover {font-style: italic;}: d.h. alle Links, über die ich mit der Maus hovere, werden kursiv.
  • Pseudoklassen, weil nicht auf ein HTML-Element an sich zugegriffen wird (a bleibt unberührt), sondern auf Element in bestimmtem Zustand

Wertigkeit: Ausnahme schlägt Regel, d.h. ID schlägt Klasse, Klasse schlägt Element

Box model, margins, and paddings

Box model:

Most HTML elements are box elements (like p)

Box model: margin, border, padding, content

  • Border: around box, making box visible
  • Padding: inside box, between border and text of paragraph
  • Margin: outside box, between boxes

Margin & padding:

  • Margin & padding accept pixels, em, percentage
  • Margin + padding can be specified: im Uhrzeigersinn, d.h. margin-top, margin-right, margin-bottom, margin-left
  • div { margin: 0px 20px 0px 10px }
  • Abkürzungen (machen Sinn auch wegen Ladezeiten) – ein Wert für alle, zwei Werte für oben/unten und rechts/links, drei Werte für oben, rechts/links, unten

Layout and individual sections

How to arrange HTML elements on web page

  • All good layouts use grid
  • Which sections do I have, which is most important (most space)

Ways of organizing my texts

  • 1-column layout: showing off some of my content (image, text, video)
  • 2-columns layout: for blog sites etc. Links to recent articles/authors can go in sidebar
  • 3-columns layout: for example for blogs: blog feed + sidebar with links + twitter feed.. anything where home page serves as portal to more information.
  • Gridded layout (Kachel): Portal-Charakter. Im Gegensatz zu 3-columns werden hier die Spalten nicht als Units wahrgenommen
  • Masonry layout: ähnlich wie gridded, aber versetzte Boxen (weniger streng)

Positionieren

Fixed: festes Positionieren

  • Hinweis "Seitenanfang" soll immer rechts-mittig zu sehen, egal wie ich scrolle.
  • Nicht für mobile Geräte geeignet
  • Bezieht sich auf Bildschirmausschnitt, bleibt beim Scrollen an gleicher Stelle, immer sichtbar
#posts {
position:fixed; - bezieht sich immer auf Ausschnitt, den ich gerade sehe
top:45% - in der Mitte
padding:10px;
}
  • Make object always sit in middle of the page: Give it position fixed, with left: 50%, and then set negative margin equivalent to half of its width.
#myobject{
position: fixed;
width: 300px;
top: 10px;
left: 50%
margin-left: -150px;
}

Absolute:

  • Bezugspunkt linke obere Ecke, padding auch nicht mit einbezogen
  • Bei Werten oben wäre das also 30px von linker oberer Ecke nach unten geschoben

Verschieben über top, left, bottom, right

#posts {
position:relative;
top:30px; D.h. 30 px von Ausgangsposition nach unten verschoben
left:0;
}

Float element

  • Float command makes block elements (sections, p, div...) float next to each other
  • {float: none}: default value
  • {float: left}/ float: right
  • {float: inherit}: inherits floating character from parent element
  • I have main content and sidebar. If I make both {float:left;}, they will sit next to each other
  • Then add margins of 30px. to show then as columns

Document flow

  • Usually, block elements stack on top of each other and inline elements display inline.
  • If I have 3 divs, and I make div1 and div2 {float-left;} and div2, they are taken out of the normal document flow - div3 does not see them, and so goes to top left side of website (appearing to be behind div1 and div2)

To avoid this:

  • not float too much, and better float few big elements and not many little (div around main, div around aside)
  • Overflow property. div {overflow: auto;}: tells html element what to do when its contents don't fit into container, to expand to necessary height
  • Clear property. div {clear: both;}: tells element whether other floating elements are allowed to float next to it (and if so, on which sides of the element), or whether element needs to be moved down below floating elements
  • So for footer: div {clear: both;}

Rules and best practices

Coding best practices

Jedes Webprojekt stellt neue Anforderungen. Deswegen kann man vor allem Herangehensweise, Prozess optimieren, um verlässlich zu guten Resultaten zu kommen. Im Web design heißt das: viel Planen, Wireframen, Persona + User Testing, Iteration and Refinement. Im Web development helfen unter anderem coding best practices: industry standards for structuring and organizing code.

Organize code

Organize by file type (css folder, image folder, js folder - dann müssen auch alle file paths daran angepasst werden, d.h. Link zu Bild immer img/....jpg). Organize code in file: indent nested elements. Google coding style guide: use two spaces instead for tabs

Style guides

Important style guides: Google and GitHub Coding StyleGuides (used by many people)

  • Always use https protocol
  • Indent by 2 spaces at a time
  • Remove white spaces, and omit optional tags (like beginning and end of html, ...) Rules for tag omission
  • Use meta charset="utf-8". Don't specify encoding of style sheets as they assume UTF-8
  • Explain code in comments: what does it cover, what purpose does it serve, why is respective solution used? (depending on complexity and purpose)
  • Action items: mark to-dos and action items with TODO
  • Be consistent, and don't make up random best practices - adapt to what others do.

Style guide for HTML

  • Use elements for what they have been created (p for paragraphs, h for headings)
  • Strictly separate structure (markup), presentation (style sheets) and behavior (scripts). Link as few style sheets and scripts as possible
  • No need for entity references (for euro, dash, ...), only exceptions for characters with special meaning like HTML like < and &
  • Use a new line for every block, list, or table element

Style guide for CSS

  • Use shorthand properties whereever possible
  • For example: p{font: 100%/1.6 palatino, georgia, serif;} - 100% font-size, 1.6 line-height, p/g/s font-family
  • Omit units after 0, and omit 0 in values (so .8em)
  • Alphabetize declarations
  • Use a space before curly brackets and after a property name's colon - font-weight: bold
  • Separate and selectors and declarations by new lines, and always start new line for each selector and for each declaration
  • Always put blank line between rules

Best practices for names

  • No spaces in anything
  • Separate words in IDs and class names by a hyphen (not underscore - also better for SEO)
  • Use ID and class names that reflect the purpose of element (and not that explain what it does, so not .button-green)
  • No CamelCase and NoPascalCase in HTML and CSS (for JavaScript ok)
  • Only use lowercase (also for hex names)
  • Be as short as possible but as long as necessary (so #nav instead of #navigation) - rule of thumb: no more then 20 characters in file name

Und sonst

  • Bei CSS nie Anführungszeichen, außer bei Schriftarten
  • Organize CSS like HTML
  • Avoid negative margins
  • Avoid fixed heights
  • Don't use too many borders
  • Don't float too much
  • Avoid floating to the right
  • If two elements share same properties, I can combine them: h1, p {color: black; font-family: sans-serif;}
