Was ist HTML?

HTML was created by Tim Berners-Lee at CERN.

  • Sharing docs via internet.
  • TBL invented HTML together with URI, HTTP, and web browser
  • Web browser: software to read HTML files via HTTP.
  • Web browsers are able to interpret html markups and present their structure accordingly.

Was machen HTML und CSS?

Dokumentenstruktur vs. Dokumentendarstellung

  • Struktur beschreibt Aufbau, Untergliederung (Überschriften, Gliederung, Absätze..).
  • Darstellung betrifft Formatierung und Layout des Dokuments.
  • CSS definiert Layout der Strukturelemente, passt auch Darstellung an unterschiedliche Ausgabemedien an (Bildschirmgröße, akustische Ausgabe)
  • CSS-Befehle können nicht vom Browser gelesen werden, nur für Gestaltung zuständig
  • Ein design.css-Sheet definiert Aussehen aller Seiten einer Website

Statisch/dynamisch

  • HTML5 und CSS können nur Oberfläche, erstellen eine statische Website
  • Statisch: kann auch Videos erhalten, heißt dass jeder User das Gleiche sieht.
  • Dynamische Elemente wären z.B. Kontaktformulare, Googlemaps.
  • Die können auf HTML-Seite zwar erstellt werden, aber nur mit externem Dienst nutzbar gemacht werden (CMS/Shopsoftware)
  • Responsive Websites passen sich an Geräte an (für mobile Geräte, Seiten minimieren usw.)

Die Grundstruktur einer Seite

<!doctype html>
<html>
<head>: Hier übergeordnete Infos, Titel und Metadaten über die Seite, die nicht im Browser zu sehen sind
<title>:: Titel der Seite, die im Reiter oben zu sehen ist, SEO-relevant
<meta charset="utf-8"> - Browser haben unterschiedliche Kodierungssysteme (UTF-8). Muss im html-head so angegeben werden. Zusätzlich noch so abspeichern. Am besten als UTF-8 ohne BOM. Nicht mit jedem Editor möglich (z.B. nicht Phase5)
</head>
<body>: Hier Navigation, (sichtbarer) Titel der Seite und Überschriften, Haupttext, Sidebars
</body>
</html>

  • Meta tags
  • Content tags

Meta tags

Meta tags tell computer things about the document and the site, not important for structure as such.

Comment tag: <!--This is my comment-->

<head>: linking to stylesheets and scripts

<!doctype html>: tells browser that this is html doc
<html> tells beginning and end of html in file </html>
<meta charset="utf-8" />: meta tag can have many attributes
<title>: what can be seen in tab
<body>: things that user sees

Content tags

Text tags

  • <h1> / <h2> <h3> / <h4-h6>: headlines
  • <p>: paragraph

Marker tags

  • <strong>: standing out, bold
  • <em>: emphasized, usually italic

Special tags

  • </br>: line break w/o paragraph (auch für extra Leerzeile)
  • &m dash;: em-dash
  • <abbr>: abbreviation or acronym
  • <hr>: erzeugt graue Linien

List tags

  • <ol>: ordered/numbered list
  • <ul>: unordered list, for most things
  • <li>: list elements
  • Für Navigation sinnvoll, OL zu machen und Nummerierung verschwinden zu lassen. Optisch ähnlich, aber für Logik wichtig, Struktur wird besser, Hierarchien und Verschachtelungen lassen sich besser darstellen

Button tags

  • <button>: clickable button

Structural tags

Very important for CSS, but no logical meaning

Div tags - define division or section of my site

  • If I have 2 columns, I mark up both columns with div tags to identify them
  • header, sidebar, etc. get div tags
  • div tags can be nested inside each other
Anwendungsbeispiel:
.stop{clear:both} – heißt, dass bei dieser Klasse aufgehört wird zu floaten
<div class="stop">

Span - einzelne Elemente markieren, die nicht mit html-Tag markiert sind

  • wie div, Textgestaltung aber innerhalb der Zeile, hat auch keine logische Bedeutung.
  • Man könnte auch theoretisch Element mit „strong“ markieren und strong eine Klasse anhängen – hätte den gleichen optischen Effekt, „strong“ wird aber von Suchmaschinen gelesen und interpretiert, ist also ein logischer Unterschied
  • Inline-Element, geht nicht über Absätze rüber

Structural HTML5 tags

Structural tags allow to be more specific about what the sections they are rendering - provide common language and logic for coders and computers. Good for SE and blind people, too. Called semantic tags - semantic web, movement to store data in a way that computers and people can understand

  • <header>: introductory content at top of page (page title, logo, navigation)
  • <nav>: navigation area
  • <main>: for main part between header, nav, sidebar, and footer
  • <aside>: sidebar - content set aside from content it's placed within
  • <section>: for sections of side
  • <article>: for articles, blog posts etc., independent content that makes sense on its own
  • <footer>: information about sitemap, contacts, copyright info
  • <audio>: defines sound content
  • <div>: for everything else, is very indescript

Images

<img>: image is a self-closing tag!

  • <img src="..." alt="..." />
  • "src" is image attribute, tells computer where image is stored (on which server, so URL address)
  • "alt" is attribute to describe what image shows, important for blind and SEO

<figure>-Markierung:

  • Image ist Inline-Element, Figure Block-Element.
  • Figure ähnlich wie div, eignet sich aber für Bilder besser, weil sie bestimmte Eigenschaften hat (div eigenschaftslos)
  • Figure macht bisschen Abstand, markiert Bild als solches (für Suchmaschinen) und erlaubt <figcaption> und andere Elemente

Bildunterschrift:

<figure>
<img src=…>
<figcaption>Weingut </figure>

Copyright-Angabe © in Bildunterschrift:

<figcaption>Weingut<cite> &-copyEsther Wahlen<figcaption>

Bildformat und -größe

  • Image file types: .jpg, .png, .gif supported by most browsers, for most others plugins used.
  • Img. not larger than 1 MB. Startseite insgesamt sollte nicht mehr als 100kb betragen. Muss schnell laden, normaler User hat 2sec. Orientierungsphase, wenn dann nicht alles geladen ist, wird er Seite verlassen.
  • Für Bildstrecken usw. auch größere Bilder möglich, aber nicht in der Vorschau

Einbettung in HTML

  • Bilddatei in Ordner abspeichern, der mit hochgeladen wird und relativen Pfad vom HTML-Doc setzen:
  • Eine Ebene hoch: <img src="../bilder/dino1.gif">
  • Zwei Ebenen hoch: <img src=“../../bilder/dino.gif">

Icon fonts:

Icon fonts are little icons. Load font like a web font and then add them in document

  • Download from fontaweseome website
  • Locate fonts in font folder, create folder "fonts" in project, and copy them there
  • Copy css files in font-awesome css folder and copy them in css folder in my project
  • Add links to fonts in each html document
  • Add icons in my navigation bar by using <i> tag, so <i class="fa fa-code"></i>
  • Define which fa-icon I want to use, for example fa-code, or any other from this cheatsheet.
  • Icons can be made responsive: made larger, wider, put instead of bullets, animated and more

Links

Links nach außen

  • <a href="URL" target="_BLANK">Descriptive text</a>: linking to external site/in new tab
  • href and target: attributes
  • link image: <a href="URL" target="_BLANK"> <img src="URL img" alt="Image" />Bild</a>

Follow/no follow

  • Verbindung meiner Seite mit Seite, auf die ich verlinke
  • Wenn ich über Seite schreibe, die ich dezidiert nicht empfehlen will (Google-Index), kann ich no-follow einfügen. Viele Kommentarspalten haben no-follow als Default. Hat auf Linkverhalten keinen Einfluss
  • <a href="..." rel="no follow">

Innerhalb des Dokument verlinken (Sprünge)

  • Element, zu dem ich springen will, ID geben: <h2 class="h2">
  • In der Inside-Navigation ID verlinken: <a href="#h2">
  • Zum Seitenanfang (immer): <a href="#">

Link zu PDF-Dokumen: <a href=“materialien/script.pdf“>Download Script</a>

Link zu E-Mail: <a href=“mailto:ich@web.de“>Kontakt</a/gt

Was macht Browser mit Links?

  • Kann .html, png, jpg, gif direkt (d.h. ohne Hilfsprogramme) darstellen.
  • Für Formate wie PDF: wenn Programm zur Darstellung im Browser vorhanden ist, lädt er das und stellt es direkt im Browser dar. Wenn nicht vorhanden, öffnet er Download-Dialog

Linkgestaltung

Darstellung als Bild, wenn ich Schaltbutton z.B. als Brief darstellen möchte: <a href=“mailto:ich@web.de“> <img src=“bilder/Brief.gif“>Kontakt</a>

  • Hover: mit Tastatur drüber: Hand. a:hover{ }
  • Active: ang drauf geklickt, noch nicht aufgerufen: rot. a:active{ }

Syntax

Block elements

Most html tags are block elements

  • Blockelemente: Blockelemente erzeugen Zeilenumbruch, z.B. <h1-h6>, <p>, <blockquote>, <div>
  • Eigenschaften will extend to whole container
  • <li>, <h>, <p>: their characteristics will extend to whole line/section

Inline elements

Characteristics defined to marked elements

  • <a>, <em>, <strong>, </br>
  • block elements never inside inline elements

Regeln

  • Inline in Block
  • Inline in Inline
  • Block in übergeordnetem Block
  • Reihenfolge: <div>, <blockquote>, &figure> übergeordnet, <p> untergeordnet.
  • D.h. figure, blockquote und div dürfen nicht in p stehen

Practical things and tools

  • In Notepad schreiben, speichern mit Endung.html
  • launch in Chrome etc.
  • In Notepad unter "View" HTML oder CSS einstellen fürs Highlighten

Ordner/Speichern

  • Name der Datei muss mit a-z anfangen, darf sonst nur 0-9, -, _ enthalten. Keine Sonderzeichen, Leerzeichen, Umlaute
  • Startseite immer als index.html abspeichern, wird vom Browser so aufgerufen
  • HTML-Dateien am besten im Roots-Ordner (und nicht in eigenem Unterordner), einfacher zu warten

Online stellen

  • Webhostdienst raussuchen, z.B. auf webhostlist.de
  • Was ich brauche: Webspace (schauen wie groß meine Seite ist), Traffic (unlimited), Domainname min. 1, Serverstatistik, ftp-Zugang.
  • Was ich nicht brauche: Datenbanken (evtl. bei WordPress: SQL (?)), php
  • Servertyp: Linux ist sicherer als Windows
  • Dienst raussuchen, anmelden
  • Übermittelt mir Servername, ftp-Nutzername, ftp-Passwort (oder Webftp: direkt über Kundenoberfläche hochladen, ohne eigenes Programm)

Hochladen auf den Server

  • Z.B. mit Filezilla
  • Dort Servername angeben, Benutzername, Passwort. Default-Port ist 21
  • Dann Dokumente, Ordner hochladen

Finishing project

  • Shipping a project: not wait until product is perfect, define it as "done" even when it is still in progress.
  • Break shipment into manageable steps. Define priorities and minimum viable product (mobile version?), and then continue shipping in small steps/increments. Set deadlines for each of them.

Quality Assurance (QA)

  • Big firms use testing software. Still important to make spreadsheet to check certain things
    • Check content (spell check, Vollständigkeit)
  • Do links and images work, is everything readable?
  • Cross-browser QA: web fonts, header + main content area + footer showing, all images displaying?

For mobile devices

  • Chrome dev tools
  • Mobile-ready: Testing website if website is mobile friendly (only working if site is online)
  • On multiple screen sizes at once: Responsive-design-testing-master - test live sites on that website, or download code from GitHub and test local site. In folder, open index.html and type in my local site
