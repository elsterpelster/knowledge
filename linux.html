Das Terminal und seine wichtigsten commands

Search, create, copy, delete things

  • dkpg -L "wine": Search application. It will list all applications with this name inside + their location
  • cp directory-x/file.html ~/Documents/directory-y/: Copy file from directory-x to director-y
  • mkdir /.../mynewfolder: create new directory
  • touch mynewfolder/index.html: create new file
  • rm -i: remove file
  • rmdir: remove/delete empty directory
  • rm -r: delete directory and files inside

Navigate and switch directories

  • pwd: print working directory: where am I?
  • cd: home
  • cd ..: one level up
  • cd Documents/skillcrush: change directory to this folder
  • cd Doc + tabkey: autofill
  • cd -: go back to previous directory
  • ls: list services in this directory
  • ls -a: list also hidden services
  • ls -d: list with details

Launch and close applications

  • nautilus /.../ or xdg-open . : open current directory (current directory: .)
  • open html.html: open with default program
  • gedit html.html: open with gedit
  • gedit *.html: open all html files in this directory with gedit
  • chromium-browser html.html: open with browser
  • open html.html: open with default program
  • gedit html.html: open with gedit
  • gedit *.html: open all html files in this directory with gedit

Administer terminal/processes

  • Ctrl+c: kill running process
  • Ctrl+z: pause running process
  • jobs: shows all background jobs on terminal
  • bg: makes last paused jobs continue in background

Git commands

Configure Git and create repositories

  • git config --global user.name "My name": Set username
  • git config --global user.email "email": Set email
  • git config --global user.name (or email): Check name
  • Go to folder and turn folder into git repository. Create 1 repository for each project. Repositories in Git contain my files and folders for projects
  • git init: create empty repository
  • git status: tells me the status of this folder in git (including untracked files), see all changes I have made - files that have been added, new files or directories etc. (good to run before adding files and committing changes)

Add and commit files

  • git add index.html: tells Git which files to add to these repositories. Add all untracked files that I want in there. Add files individually (usually best)
  • git add .: add all files in current folder
  • git add -A: add all files in repository
  • git commit -m: commit files, like save command, and I can also add comments:
  • git commit -m "my fist commit
  • git log: tells me history of commits and comments (and who committed at what time)
  • After that, edit code files until I am at a good stopping point, commit files using git add command (for all files) with comment what I changed

Some git commands

  • git diff path/.../index.html: compares local file with version of my most recent commit
  • git pull: commits down from remote repository to local repository
  • git push: pushes local commit to remote
  • git rm: removes file or directory from repository
  • git help: list of common commands

Git branches

  • Idea: We have codebase (trunk - master branch) with thousands of files and 10 developers working on it
  • If I make changes to a codefile, do not make this on trunk, but create branch (parallel version of code, like a copy) and edit this branch. If these changes make sense, merge branch with trunk, so integrate branch back into master branch.

How to create a branch

  • Inside my folder "project", I have folder with codefiles.
  • git branch: shows me existing branches and tells me branch that I'm currently on, usually master branch
  • git checkout -b mynewbranch: create new branch called "mynewbranch" and switches to it. New branch is exact copy of master branch. Changes I make and commit to branch will not show in codebase (so codefiles I find in folder), but only in branches. Only if I merge branch, they will show

Make changes to branch

  • git status: If I have created branch "mynewbranch", check if I am on this branch. Make changes to any file now and save
  • If I want to test out these changes in browser, make sure I am on this branch and then open it in browser. If I want to test it out against master trunk, switch to master and open it in browser

Merge changes

  • git checkout master: checkout master branch and switch to it. To merge, I need to be on branch that I want to merge into!
  • git merge mynewbranch: merge changes of new branch into branch I am currently on (so master branch)

Merge conflicts

  • If several people edit same line of code on several branches and then try to merge, that's a merge conflict, because which changes are authoritative?
  • If I want to merge and git tells me that there is a merge conflict in certain file, I need to open file. Git adds code where merge conflict is (everything in top section is from master branch. Everything from lower section is from experiment branch). I need to decide then and delete code added by git
  • Then add and commit

Git branch commands

  • git checkout master: switch back to master (and not create new branch)
  • git checkout -b branchname: create new local branch
  • git checkout branchname: checks out existing bands
  • git branch -a: shows all existing branches and highlights current branch ("remote" refers to external repository)
  • git branch -d branchname: deletes specified branch
  • git merge branchname: merges specified branch into current branch

GitHub

Set up: connect computer to GitHub

If I want to access repository, GitHub needs to verify that I have access to it

Access repository with special link, which defines how data is transferred over the internet. Process is called transfer protocol. Identity is verified depending on type of transfer protocol I use

  • git clone https://github.com/skillcrush/wordpress.git would be access via https protocol. Then I need to enter my username and password every time I push or pull code
  • git clone git@github.com/skillcrush/wordpress.git is access via SSH protocol: then I do not need to enter credentials

Verification with SSH keys.

  • Every time I push and pull, GitHub compares private or public keys
  • Keys in general stored are in special folder called .ssh in home directory. If I don't have one, create .ssh folder in home directory
  • ls -la: check if .ssh repository has a public key: if there is some id_dsa.pub or id_rsa.pub, I have a public key

If I do not have a key, create one

  • ssh-keygen -t rsa -C "myemail@email.com"
  • Press enter, generate password (24K)
  • ssh-add ~/.ssh/id_rsa - add key to ssh agent, then I won't have to enter password every time my key is accessed

Add public key to GitHub account

  • pbcopy < ~/.ssh/id_rsa.pub: copy public key
  • Go to GitHub account, Settings, click on "SSH and GPG keys" and click "New SSH key". Give title "elsterpelsters thinkpad" and paste contents of public SSH key
  • Click "Add SSH key" and insert public key (that I find in .ssh folder)
  • cd ~/.ssh and run ls -la: check SSH keys in this folder. It should show either id_dsa.pub or id_rsa.pub
  • Now computer should be able to connect to GitHub
  • GitHub has command with which I can verify that I added keys correctly: in terminal, type ssh -T git@github.com. Type yes and hit enter (will say that it does not provide shell access, which is fine)

How GitHub is organized

Every repository has its own page, where I see all files and folders that belong to project

  • I always see most recent version, but can also see history by clicking on "history" on top right.
  • Blue icon on the right hand of file shows details. Lines highlighted in red where removed, lines highlighted in green where added
  • Settings: add collaborators to repository, change visibility
  • Search box: search codefiles

GitHub repositories

On GitHub, repositories can be either private or public.

  • Private only available to people I have granted access to via this repository's setting page.
  • Public repositories are available to everyone and can be searched and edited on GitHub by everyone html tags

Create new repository

  • Next to my account on top, click + and create new repository. Name should be lowercase and descriptive and set to public or private (public are free). No readme.
  • Connect to local respository: switch to SSH protocol
  • Add GitHub repository as remote origin for Git repository: check if I already have remote origin: cd to my project and type git remote -v
  • If nothing shows up, type git remote add origin git@github.com elsterpelster/knowledge.git. Check git remote -v again, should show up

Push up code

  • Push up code for first time and set myself as master for this code (so I will be informed about any changes on that code): git push -u origin master. Now all my code should be added
  • Push changes: If I now make any changes to my code and add and commit them, I have to push them to remote repository
  • First, pull code for any changes that may have been made (get habit of doing that)
